# 商队Rogue玩法Mod（暂用名）
## 玩法介绍大纲

全面战争游戏原本的玩法包含大量的模拟经营环节与大地图战略决策环节，每局游戏时间较长；同时，游戏一旦经过前期就会快速进入玩家军队过于强大、大地图无脑填色的游戏环节中。具体来说，本Mod的设计初衷，是针对原版游戏模式的如下问题提出解决方案：
1）原本游戏经营环节过长，对于不想体验这部分的玩家会重复枯燥；
2）很多相对弱势的中期兵种，由于“性价比”较低，玩家倾向于干脆不召集这部分兵种类型，在多数游戏环节中没有存在感；
3）玩家后期全是清一色的高级兵，而电脑的部队则倾向于杂而不精，没有足够实力的对手进行匹配
针对以上问题，本Mod期望提炼出一个以“商队闯关”为主的玩法，类似原版游戏中的震旦商队。玩家无法自主招兵或建造建筑，而是通过一连串的触发事件与选项来推进游戏进程、参与随机战斗、取得随机奖励。战斗与奖励都是通过事件窗的形式来触发，类似原版的任务战斗或者是奖励事件。

## 主要玩法流程

* 玩家军队：玩家讲拥有一支类似与“商队（或车队）身份”的军队，这个军队无法被玩家控制移动，但是玩家可以给军队中的领主/英雄单位进行正常加点/更换装备的能力。这部分军队就和原版商队系统中的部队一模一样。
* 推进流程：通过一个玩家在回合界面可以通常点击到的按钮进行触发，比如震旦的罗盘按钮或高精灵的凯恩之剑按钮旁边再创建一个新的Mod事件启动按钮。玩家点击按钮后，触发推进流程事件。触发流程事件分为四种，包括：
  * 选择目的地事件：给玩家提供2~3个大地图上的标志性城市或地区作为目的地，允许玩家的军队模拟移动到选择的地方；
  * 战斗事件：根据玩家所处的位置，给玩家生成一个对手军队并进行战斗事件，正常触发战斗，玩家可以选择自动战斗或者手动战斗；
  * 单位奖励事件：根据玩家所处的位置与原本的派系类别，给玩家生成3个随机兵种选项，允许玩家获得，事件类似于商队事件中的获取部队事件；
  * 装备奖励事件：给玩家生成2~3个随机的装备或魔法道具选项，允许玩家选取一个获得，事件类似于高等精灵派系的“瓦尔之祷祭礼”；
* 游戏结束：当玩家连续再多次战斗事件中落败后，判定玩家失败

## 架构设计思路

为了实现游戏玩法，可以考虑参考如下的架构设计思路。这里并非作为硬性规范，仅作为一种参考，可根据实际情况进行沟通与调整。

* 主流程模块：初始化mod，配置相关参数、概率等，在玩家进入游戏后调整相关功能与模块的可用性，并调用军队模块的军队生成给玩家生成一个初始军队；
* 玩家状态模块：记录玩家的军队，派系，战胜失败次数，连续失败次数，已经完成的战斗事件数，玩家单位的所属地区；
* 虚拟地图模块：通过一个给定的外部文件读入一个节点与边表作为地图。这个文件来定义可以使用的城市节点与彼此之间的关联；同时，文件还会定义节点的所属派系；
* 事件窗模块：用来给玩家打开队对应的事件窗并处理相关逻辑。相应玩家点击Mod事件启动按钮或玩家选择的选项。由于全面战争在事件窗下无法给军队调整配置与技能，持续不断的事件触发会让玩家无暇调配，所以所有事件窗的最后一个选项要固定为暂时关闭事件窗。当玩家重新点击Mod事件启动按钮，就会还原原本的事件窗，从而继续推进流程。这个按钮如果找不到合适的实现参考，可以参考SummonServant&GenericFramework参考Mod中的SummonServant按钮。
  * 事件生成器：根据玩家当前的状态生成事件。生成的顺序是，1.单位奖励事件，2.战斗事件，3.（一定概率出现）装备奖励事件，4.目的地行进事件，然后回到1.单位奖励事件，每完成一个事件直接进入下一个事件，循环往复。生成时，需要调用事件难度管理模块来取得所需相关参数。如果玩家关闭事件窗，重新点击事件窗触发时，需要还原被关闭的，而不是直接进行下一步。
  * 单位奖励事件：调用军队模块的奖励单位生成能力，来生成2~3个单位奖励选项（此外还有一个暂时关闭事件窗的选项）生成单位中，固定1个是玩家当前所处位置的派系，另外的1~2个是玩家自己原本派系的单位。(TODO:能否使用类似帝国元帅马库斯的帝国援军或食人魔营地提供的招募池系统？不是直接给军队提供单位，而是给玩家提供对应单位的1或n次招募数量，英雄单位能通过这种方式来配置吗？如果不可以，那么就需要进行更加详细的沟通设计了。)
  * 目的地行进事件： 需要调用虚拟地图模块。根据玩家当前所处的节点，在虚拟地图上随机选择本节点的2个邻接节点，作为可以选择的行进位置；此外，停在原地不动固定作为第三个选择。如果玩家选择某个地点选项后，将玩家的部队转移到对应坐标并提供视野。
  * 战斗事件：触发战斗，基本可以照抄商队系统触发的遇敌事件，不过不提供避战选项，只有进行战斗的选项。并且，触发战斗不是真的使用玩家的部队进行战斗，而是创造一个玩家部队的完全一模一样的拷贝来让玩家控制参与这次战斗，这只克隆部队负责战斗，战斗结束后直接删除(这里其实只是为了让玩家在战斗中的所有军队损失不被保留，每次参加战斗都是完全满状态，如果有更合适的实现方法也可以)。战斗后，将玩家原本所有的单位/人物提高一个等级（英雄与领主会因为升级而获得1技能点），如果满级就不要继续升级了。
  * 装备奖励事件：调用随即装备模块，随机选择3个左右装备作为可以获得的选项，玩家选择后可以得到装备。基本可以照抄高等精灵的祭礼“瓦尔之祷”。
* 军队模块：军队模块主要用于生成军队或奖励单位。为了能够量化生成的单位的能力，拟采用“价值分”来作为量化标准。希望能够采用原本游戏用于“多人对战”模式的兵种的金钱价格（TODO:能否读到？）作为价值分，如果无法读取或使用，可以再继续讨论其他方案；
  * 军队生成：根据给定的“价值分”与派系，随机生成一个对应派系的总价值分接近的军队，包括1名本派系的随机领主（可以是传奇领主），以及19个以内其他单位，可以包含英雄单位（事务官单位）。具体来说，价值分接近指可以在总价值分的0.8倍到1.2倍以内。
  * 奖励单位生成：根据给定的“价值分”与派系，随机抽取一个价值分近似的奖励单位的兵种类型，可以是英雄单位（事务官单位）；价值分近似指在给定价值分0.8~1.2倍之间。
* 随机装备模块：维护一个所有装备的价值字典，根据给定的价值层级（可以按原本道具的稀有级别划分），随机选取装备返回。
* 事件难度管理模块：根据玩家已经完成的战斗事件数（无论是否胜利），设定当前事件应有的敌人军队总价值分、玩家获得奖励单位的价值分、以及随即装备的价值层级


## MVP目标

第一版只验证以下流程：
1. 玩家开始新战役后获得一支脚本生成的测试军队；
2. 玩家可以通过一个按钮或事件入口推进流程；
3. 推进后依次触发：单位奖励事件 → 战斗事件；
4. 战斗胜利后记录完成次数，并再次进入奖励事件；
5. 暂不实现虚拟地图、装备奖励、复杂难度曲线、失败结算。

同时，第一版仅支持震旦派系先完成验证；后续再进行其他派系尝试。

在单位奖励事件的第一版实现中，可以先做一个简单的实现：
- 如果商队军队未满 20 队，奖励单位直接加入军队；
- 如果已满 20 队，则在事件对话中提示玩家需要手动解散部分部队，并提供关闭事件窗的按钮。之后玩家再重新点击Mod按钮回到原本的奖励事件
- 第一版不支持英雄单位作为奖励；
- 第一版不使用招募池。

第一版实现边界：
- 暂不支持多人战役；
- 暂不保证与大修 Mod 兼容；
- 暂不支持所有派系；
- 暂不支持海上节点；
- 暂不支持混沌裂隙、传送、地下道等特殊移动；
- 暂不支持英雄单位奖励；
- 暂不支持自定义战斗地图；
- 暂不处理玩家主动招募和建筑系统，第一版仅作为附加玩法测试。


## 流程状态机

状态包括：
- INIT：初始化
- UNIT_REWARD_PENDING：等待玩家选择单位奖励
- BATTLE_PENDING：等待玩家进入战斗
- EQUIPMENT_REWARD_PENDING：等待玩家选择装备奖励
- DESTINATION_PENDING：等待玩家选择目的地
- PAUSED：玩家暂时关闭事件窗
- GAME_OVER：挑战失败

玩家点击 Mod 按钮时：
- 如果当前状态为 PAUSED，则恢复上一次未完成事件；
- 如果当前状态为空，则根据 completed_battle_count 生成下一个事件；
- 如果当前状态为 GAME_OVER，则显示失败总结事件。


## 玩家状态字段

- player_faction_key：玩家派系 key
- caravan_force_cqi：商队军队 CQI
- caravan_character_cqi：商队领主 CQI
- current_node_key：当前虚拟地图节点
- current_event_type：当前待处理事件类型
- current_event_seed：当前事件随机种子
- completed_battle_count：已完成战斗次数
- victory_count：胜利次数
- defeat_count：失败次数
- consecutive_defeat_count：连续失败次数
- selected_route_history：已访问节点列表
- reward_history：已获得奖励列表
- difficulty_level：当前难度等级
- run_status：running / paused / failed / completed


## 装备奖励规则：
- 奖励直接加入玩家派系物品库存；
- 第一版只奖励普通装备与魔法物品；
- 不直接自动装备；
- 不奖励随从、旗帜、特殊任务物品；
- 装备池按稀有度分为 common / uncommon / rare / unique；
