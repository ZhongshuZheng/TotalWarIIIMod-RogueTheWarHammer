# AdamRogue 阶段E实现规范

## 文档定位

本文档定义并指导阶段 E 的最终实现规范，进行数值与平衡方向功能设计。

用途：

- 固定平衡目标
- 提供开发诉求

## 阶段 E 的核心目标

阶段 E 不是简单“把数值调大”，而是建立一套可持续的 run 难度曲线。

这套曲线至少要同时控制：

- 敌军总价值分
- 玩家单位奖励价值分
- 敌军单位数量
- 装备奖励层级开放
- 精英战压力峰值

结论：

- 阶段 E 应被理解为“完整游戏平衡设计阶段”，而不是零散修补。


## 阶段 E 正式定义的 8 个平衡模块

### 1. 初始双方总价值分

当前建议：

- 玩家：`4500`
- 敌军：`4000`


- `1 个随机非传奇领主 + 随机的单位` 抽取满足生成部队的weight
- 生成部队时，玩家与敌军部队的领主本身得分需要被扣除

### 2. 每循环敌军总价值分增量

按照如下方式定义：

- 1~5轮敌人开始生成前，总分数每轮增长 500
- 6~10轮敌人开始生成前，总分数每轮增长	600
- 11~15轮敌人开始生成前，总分数每轮增长	950
- 16~20轮敌人开始生成前，总分数每轮增长	950
- 21~25轮敌人开始生成前，总分数每轮增长	1000
- 26~30轮敌人开始生成前，总分数每轮增长	1100
- 31轮以后敌人开始生成前，总分数每轮增长	1500

- “轮”指的是：每次移动目的地事件完成过后（推进到奖励单位事件之前），作为循环+1轮。游戏开始初始状态为第1轮，第一次移动目的地完成后变为第2轮

实现实例：

enemy_value[1] = 4000

从第 1 轮开始，每轮在上一轮基础上增加对应区间的 growth：

enemy_value[cycle] = enemy_value[cycle - 1] + growth(cycle)

growth(cycle):
- 1~5：500
- 6~10：600
- 11~15：950
- 16~20：950
- 21~25：1000
- 26~30：1100
- 31+：1500

其他说明：

- 按实例设计，在第一轮战斗时，敌人总分已经到达了4500
- 敌人军队的生成必须有一名领主（暂不考虑传奇领主，暂不考虑英雄），其他兵种只要是相同派系，不做任何限制

### 3. 每循环玩家单位奖励价值分

|           | 1~5轮奖励 | 6~10轮奖励 | 11~15轮奖励 | 16~20轮奖励 | 21~25轮奖励 | 26~30轮奖励 | 31~轮奖励 |
|-----------|-----------|------------|-------------|-------------|-------------|-------------|-----------|
| 最低值     | 300       | 300        | 700         | 900         | 1200        | 1500        | 1500      |
| 最高值     | 700       | 900        | 1200        | 1500        | 2000        | 2500        | 5000      |


它表示：

- 在第x轮时，玩家单位奖励候选的目标价值区间是多少

其他说明：
- 奖励单位生成，3个选项不能彼此重复。如果在价值区间内没有合适单位，向下找价值分最接近的单位
- 奖励单位不可选择领主，暂时不包括英雄（事务官），其他兵种类型不做限制，如传奇单位、Regiments of Renown、特殊招募单位，炮兵，巨兽，飞行单位均可（这里注意检查一下之前生成的python脚本和lua脚本是否有做了多余的限制）

### 4. 装备层级开放区间

- `1-5`：Common
- `6-10`：Common + Uncommon
- `11-15`：Common + Uncommon + Rare
- `16-20`：Uncommon + Rare + Unique / Legendary
- `21-25`：Rare + Unique / Legendary + Crafted
- `26以后`: Rare + Unique / Legendary + Crafted 

精英战额外规则：

- 精英战胜利奖励固定从最高层抽取

额外调整：

- 暂时将Unique / Legendary物品中，传奇领主/英雄的专属装备全部移除


### 5. 精英战机制

当前方案：

- 第 `5, 10, 15, 20, 25, 30` 轮为精英战
- 该轮敌军总价值分临时提高到 `1.25` 倍
- 该轮胜利后的装备奖励固定从“最高层级”里选择

说明：

- 直接无视进度，提前掉落最终级装备
- 精英战失败与正常战失败一样，不做单独处理


### 6. 敌军单位数量曲线

当前方案：

- 敌军数量尽量满足 `>= 2 + 已循环数`
- 敌军数量上限为 `20`
- 第 11 轮及之后至少达到 `12` 队
- 第 19 轮之后尽量保证 `20` 队满编

设计原则：

- 中后期优先保证“部队数量成型”
- 具体敌人生成价值分的区间可以在 0.9 ~ 1.0 * 总价值分，如果实在无法满足，可以到 0.85 * 总价值分


### 7. 难度修正

难度修正：

Easy:
- 敌军价值 x0.85
- 玩家奖励价值 x1.15
- 精英战倍率 x1.15

Normal:
- 敌军价值 x1.00
- 玩家奖励价值 x1.00
- 精英战倍率 x1.25

Hard:
- 敌军价值 x1.15
- 玩家奖励价值 x0.95
- 精英战倍率 x1.30

Very Hard:
- 敌军价值 x1.30
- 玩家奖励价值 x0.90
- 精英战倍率 x1.35

第一版只做 Normal，只把配置位留出来。


### 8.单位生成权重

```
def derive_unit_weight(unit_value: int) -> int:
    if unit_value <= 350:
        return 8
    if unit_value <= 550:
        return 7
    if unit_value <= 800:
        return 6
    if unit_value <= 1100:
        return 4
    if unit_value <= 1400:
        return 3
    if unit_value <= 1700:
        return 2
    return 1
```

玩家初始部队与敌人部队生成会依据此权重进行。
对于敌人部队，先按此权重抽取，如果距离价值分比较远，会把高权重（意味着低价值）单位逐一替换成更高级的随机单位，直到满足价值约束或已经全都是最高层级或本派系没有跟高级单位了。

## 实现顺序

1. 先做“平衡配置模块”与 cycle 状态落地。
2. 再接敌军价值曲线和精英战判定。
3. 再接单位奖励价值区间与去重逻辑。
4. 再接装备稀有度开放和专属装备过滤。
5. 最后调敌军数量曲线与初始玩家军队随机化。
